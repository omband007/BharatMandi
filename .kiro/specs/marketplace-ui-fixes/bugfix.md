# Bugfix Requirements Document

## Introduction

The Marketplace.html page currently has several UI/UX issues that negatively impact the user experience when browsing and viewing listing details. These issues include:

1. Listings table not displaying primary images as profile pictures
2. Incorrect labeling of "Purchase Details" section
3. Poor layout with vertical stacking instead of side-by-side view
4. Images not displaying correctly in listing details
5. Missing full-screen image viewing functionality with navigation

These issues prevent users from effectively browsing listings and viewing product images, which are critical for making purchase decisions in a marketplace context.

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN a user views the listings table in Marketplace.html THEN the system does not display primary images as profile pictures for each listing

1.2 WHEN a user views the listing details section THEN the system displays the heading as "Purchase Details" instead of "Listing details"

1.3 WHEN a user views the Marketplace.html page THEN the system displays "Listings" and "Purchase Details" sections stacked vertically (up and down) instead of side by side
